Providers used by rules_ll.

LlCompilationDatabaseFragmentsInfo#

LlCompilationDatabaseFragmentsInfo(cdfs)

Provider containing compilation database fragments.

FIELDS

Name

Description

cdfs

A depset containing compilation database fragments. Assembling the compilation database fragments into a compile_commands.json file produces a compilation database for tools like clang-tidy.

LlCompilationDatabaseInfo#

LlCompilationDatabaseInfo(compilation_database)

FIELDS

Name

Description

compilation_database

A compile_commands.json file containing a compilation database.

LlInfo#

LlInfo(exposed_angled_includes, exposed_defines, exposed_hdrs, exposed_includes, exposed_bmis)

Provider returned by ll targets.

FIELDS

Name

Description

exposed_angled_includes

A depset containing angled include paths. These include paths are carried to direct dependents.

exposed_defines

A depset containing defines. These defines are carried to direct dependents.

exposed_hdrs

A depset containing header files. These header files are carried to direct dependents.

exposed_includes

A depset containing include paths. These include paths are carried to direct dependents.

exposed_bmis

A depset containing precompiled module interfaces. These interfaces are carried to direct dependents.
